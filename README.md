## README Standard For Linked Projects

Each linked project README should answer the same questions quickly:

1. **Problem:** What decision or workflow was broken?
2. **Scope:** What the project includes and what is intentionally out of scope.
3. **Architecture:** Data flow, model/retrieval/agent flow, API surface, and dashboard/reporting layer.
4. **Result:** Measurable outcome, validated behavior, or decision artifact.
5. **Run It:** Minimal setup commands, demo path, and smoke-test command.

This keeps the portfolio homepage concise while giving recruiters and hiring managers a clear deeper-read path through GitHub.
